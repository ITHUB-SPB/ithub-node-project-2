# Сопровождение курса "Основы Node.JS"

## (КТ4 + КТ5 + КТ12) Парсер новостных RSS-лент

Приложение представляет собой утилиту для получения новостей с rss-лент новостного сайта "Ведомости".

Пользователь выбирает интересующие его рубрики и расписание, по которому далее получает выгрузки новостей в pdf-формате.

## Требования

Приложение состоит из четырёх подсистем:

- консольное взаимодействие,
- парсинг,
- рендеринг,
- шедулинг.

#### Консольное взаимодействие

Для реализации консольного взаимодействия используется библиотека `Inquirer.js`.

При первичном запуске пользователь выбирает интересующие его рубрики и расписание, эти данные сохраняются в файле `storage/settings.json` следующего вида:

```json
{
  "rubrics": ["business-energy", "politics"],
  "schedule": {
    "daysOfWeek": [0, 3, 5],
    "time": ["17:30"]
  }
}
```

При последующих запусках пользователя встречает информационное сообщение о его текущих настройках, и при желании он может изменить их.

После настройки запускается фоновый режим.

#### Парсинг

##### Парсинг списка рубрик

Парсинг списка рубрик происходит со страницы [https://www.vedomosti.ru/info/rss](https://www.vedomosti.ru/info/rss), его результаты сохраняются в файл `storage/rubrics.json` в следующем виде:

```json
{
  "rubrics": [
    {
      "title": "Бизнес",
      "id": "business",
      "link": "https://www.vedomosti.ru/rss/rubric/business"
    },
    {
      "title": "Бизнес — ТЭК",
      "id": "business-energy",
      "parentId": "business",
      "link": "https://www.vedomosti.ru/rss/rubric/business/energy"
    }
  ],
  "last_modified": "2026-03-01T20:46:07.571Z"
}
```

Для парсинга используется `node-html-parser`

Перед непосредственной загрузкой новостей проверяется время последней модификации, парсинг производится только если данные устарели. Время устаревания данных - сутки (но при разработке используйте что-то более удобное).

##### Парсинг новостей

Парсинг новостей происходит по расписанию на основе рубрик из `storage/settings.json` и ссылок на их ленты в `storage/rubrics.json`.

#### Рендеринг

Результаты выгружаются по выбранному пользователем расписанию в `pdf`-файл с названием вида `vedomosti_yyyy_mm_dd_hh.pdf`. Для формирования отчета первично используется шаблонизатор `ejs`, который рендерит полученные данные в `html`, а сохранение в `pdf` происходит посредством `puppeteer`.

#### Фоновый режим

После запуска приложение с интервалом в минуту проверяет время и при достижении времени выгрузки запускает последовательность операций:

1. Парсинг рубрик (при устаревании)
2. Парсинг требуемых rss-лент
3. Рендеринг HTML
4. Генерация и сохранение PDF

В консоли при этом отображается прогресс и сообщение об успехе или неудаче.

## Критерии оценивания

_Максимальный балл - 24_

**Консольное взаимодействие (до 4 баллов)**

1.1 Реализована проверка на то, впервые ли пользователь запускает программу (1 балл)

1.2 При первичном запуске пользователю предоставляется выбор интересующих его рубрик и расписания. (2 балла)

1.3 При повторных запусках пользователя встречает информационное сообщение о его текущих настройках, и при желании он может изменить их. (1 балл)

**Хранилище (до 4 баллов)**

2.1 Реализована запись рубрик в `storage/rubrics::writeRubrics` (1 балл)

2.2 Реализована запись настроек в `storage/settings::writeSettings` (1 балл)

2.3 Реализовано чтение рубрик в `storage/rubrics::loadRubrics` (1 балл)

2.4 Реализовано чтение настроек в `storage/settings::loadSettings` (1 балл)

**Парсинг (до 6 баллов)**

3.1 Реализовано получение новостей по ссылкам и их парсинг из `XML` (2 балла).

3.2 Реализовано получение информации о рубриках и их парсинг из `HTML` (2 балла).
Перед запуском лоадера происходит проверка свежести данных, и получение данных
происходит только в случае если они устарели, и после этого данные о последнем
изменении обновляются (2 балла).

**Рендеринг (до 7 баллов)**

4.1 Шаблонизатор генерирует корректный HTML с выгрузкой новостей на основе переданного объекта (2 балл).
Использует partial-наследование (1 балл) и стилизацию (1 балл).

4.2 Результаты сохраняются в pdf с использованием библиотеки `puppeteer` (2 балла). Файл имеет название вида `vedomosti_yyyy_mm_dd_hh.pdf` (1 балл).

**Шедулинг (до 3 баллов)**

5.1 Программа работает в фоне, корректно обрабатывает интервалы и запускает последовательность операций. (2 балла)

5.2 Работа корректно (грациозно) завершается по `ctrl+c` (SIGINT), без подвисания и артефактов. (1 балл)
